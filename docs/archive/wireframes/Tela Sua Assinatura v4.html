  • Spinner: 14×14, border 2px white/35%, top brand white, animation spin .8s linear infinite.
  • aria-label="Salvando alteração" v3 + role="status" no spinner + aria-busy="true" no botão. Leitores de tela anunciam o estado.

Por que Cancel ativo? v3

  • Cancelar fetch em voo é trivial com AbortController. Desabilitar era fricção sem reciprocidade.
  • Cliente que apertou Confirmar e percebeu erro ("errei a quantidade!") não fica preso esperando — honra a ação dele.
  • Race: se o POST retornar 200 antes do abort, a tela trata como sucesso normal (ignora o cancel). O usuário vê o toast de sucesso — estado consistente com o backend.

Duração esperada

  • POST típico ~400-800ms. Spinner sempre visível pelo menos 300ms pra evitar piscar.
  • Acima de 3s sem resposta: timeout → erro de rede (tela 8).

Anatomia do toast de erro

warning-bg · warning-border · warning-text
  • Posição: absolute top:80px, left/right:16px. Acima do modal-overlay (z-index 60 vs 40).
  • Ícone info circular 18×18 stroke 1.8 + texto 13px.

Tipos de erro que caem aqui

  • Timeout (sem resposta > 3s)
  • HTTP 5xx (problema do servidor)
  • Conexão caiu / sem internet
  • Validação de backend (raro nessa tela — composição já é validada no front)
